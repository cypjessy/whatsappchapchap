## 🎯 Next Steps to Get Your APK

### Critical: Deploy Your Backend First!

Your app uses Next.js API routes that require a server. You MUST deploy your backend before building the Android app.

#### Option 1: Vercel (Recommended - Easiest)
```bash
npm install -g vercel
vercel
```

#### Option 2: Railway
- Connect GitHub repo to Railway.app
- Auto-deploys on push

#### Option 3: Render
- Connect GitHub repo to Render.com
- Free tier available

### After Deployment:

1. **Create `.env.local` file:**
   ```bash
   cp .env.example .env.local
   # Edit and set NEXT_PUBLIC_API_URL to your deployed URL
   ```

2. **Build the app:**
   ```bash
   npm run build:android
   ```

3. **Open in Android Studio:**
   ```bash
   npm run open:android
   ```

4. **Build APK in Android Studio:**
   - Wait for Gradle sync
   - Build → Build Bundle(s) / APK(s) → Build APK
   - Find APK at: `android/app/build/outputs/apk/debug/app-debug.apk`

5. **Install on your device:**
   ```bash
   adb install android/app/build/outputs/apk/debug/app-debug.apk
   ```

## ⚠️ Important Notes

### API Routes
Your app has many API routes (`/api/*`). These CANNOT work in a static mobile app without a backend server. You have two options:

1. **Deploy backend separately** (Recommended)
   - Deploy to Vercel/Railway/Render
   - Set `NEXT_PUBLIC_API_URL` to point to it
   - All API calls will route to your deployed backend

2. **Refactor to client-side only** (Complex)
   - Remove all API routes
   - Call Firebase/Evolution API directly from client
   - Requires significant code changes

### Firebase Configuration
For Firebase to work in Android:
1. Add Android app in Firebase Console
2. Add package name: `com.whatsappchapchap.app`
3. Download `google-services.json`
4. Place in `android/app/` directory
5. Rebuild the app

### Evolution API
Ensure your Evolution API instance is:
- Accessible from mobile networks
- CORS-enabled for your app
- Using HTTPS (required for Android)

## 🐛 Troubleshooting

### Common Issues:

**"Cannot connect to API"**
- Check `NEXT_PUBLIC_API_URL` is set correctly
- Verify backend is deployed and running
- Test API endpoints in browser first

**"Gradle sync failed"**
- Open Android Studio
- File → Invalidate Caches / Restart
- Ensure Java JDK 17 is installed

**"App crashes on startup"**
- Check logcat in Android Studio
- Verify all environment variables are set
- Ensure Firebase config is correct (if using)

**"Images not loading"**
- Use absolute URLs
- Check image hosting allows mobile access
- Verify network permissions

## 📦 Publishing to Play Store

1. Generate signed AAB:
   ```bash
   cd android
   ./gradlew bundleRelease
   ```

2. Create keystore (first time only):
   ```bash
   keytool -genkey -v -keystore my-release-key.jks -keyalg RSA -keysize 2048 -validity 10000 -alias my-alias
   ```

3. Configure signing in `android/app/build.gradle`

4. Upload AAB to Google Play Console
